# Finite Fault Clustering & Plotting Pipeline

本リポジトリは、PDTIの計算結果を入力として、断層モデルの階層型クラスタリングによるクラスターの分割と、GMT6（Generic Mapping Tools）を用いた多様な可視化を行うための解析パイプラインです。
本リポジトリは、すべてシェバン（Shebang）が設定されています。システム環境変数（PATH）を通しておくことで、PDTIの結果が格納された任意の作業ディレクトリから直接コマンドを呼び出して実行可能です。

## リポジトリの構成

本リポジトリは、処理の流れに沿って2つのディレクトリに分割されています。

- `1_clustering/`
  - PDTIのインバージョン結果からクラスタリングに必要なスナップショットデータを抽出し、特徴量計算および階層型クラスタリングを実行するPython/Bashスクリプト群です。サーバー用の高速計算モジュールも含まれています。
- `2_plotting/`
  - インバージョン結果およびクラスタリング結果をGMT6で可視化するためのBashスクリプト群です。図面を描画するためのデータ成形用Pythonスクリプトや、Fortran-Python連携ラッパー（`f90wrap`）、GMT用フォーマット変換ツールもここに格納されています。

## 実行の前提条件

本ツール群は、PDTIによるインバージョン計算が完了し、以下のファイルがカレントディレクトリ（作業ディレクトリ）に存在することを前提として動作します。コードをクローンしたのち、**PDTIの計算結果が格納されたディレクトリ（results_xxxxx）上でこれらのスクリプトを実行**してください。

**【必須となるPDTI出力ファイル例】**
- `fort.40` （断層パラメータや計算設定情報）
- `mrf.dat` （モーメント解放関数データ）
- `pddis.dat`, `pdtdis.dat` （すべり量・破壊伝播データ）
- 波形データディレクトリ（`Wave_cp/` または `wave.obs_f/` など ※波形比較図を描画する場合）

## 全体作業フロー

1. PDTIを実行し、結果が保存されたディレクトリに移動する。
2. `1_clustering/snapper.bash` (または `snapper_mf.bash`) を実行し、データ成形を行う。
3. `1_clustering/align_fault.py` を実行し、節面選択（推奨）を行う。
4. `1_clustering/autttto_clustering.py` を実行し、対話型モードでクラスタリングを行う。
5. `2_plotting/` 内の各GMTスクリプトを実行し、結果を可視化する。
