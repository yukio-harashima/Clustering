# 1_clustering: データ抽出およびクラスタリング実行モジュール

このディレクトリには、PDTIの計算結果から時間ごとの破壊プロセス（スナップショット）を抽出し、階層型クラスタリングを用いてデータの分類・特徴抽出を行うためのコード群が格納されています。


## 収録ファイル・ディレクトリの詳細解説

### 1. `snapper.bash` / `snapper_mf.bash` (クラスタリング用 データ成形スクリプト)
クラスタリングの起点となるBashスクリプトです（単一断層面用が `snapper.bash`、複数断層面用が `snapper_mf.bash`）。
PDTIの出力結果（`fort.40` や `mrf.dat` など）を読み込み、内部でFortranのデータ成形ツール（`fgensnap_y` 等）を呼び出して、クラスタリングの入力となるスナップショットデータ（`snap_y.dat` や `snap.dat` など）を生成します。

### 2. `align_fault.py` (節面選択)
抽出されたスナップショットデータに対して、クラスタリングの精度を向上させるための事前処理を行うPythonスクリプトです。

メカニズム解の2つの節面（Nodal planes）のうち、コード内で設定した面に近い面を第一面として選択・入れ替え処理を行います。

### 3. `autttto_clustering.py` (クラスタリング実行司令塔)
本ディレクトリのメインプログラムとなるPythonスクリプトです。

対話型のインターフェースを備えており、ユーザーがターミナル上で変数の選択方法、PCA（主成分分析）の適用有無、距離計算メトリクス（コサイン類似度やユークリッド距離など）を選択しながら、階層型クラスタリングを実行します。

出力として、各データポイントのクラスター所属情報や、クラスターごとの代表モデル（重心データ）が記載されたファイル（`clusteringSnap2.dat` など）を生成します。

### 4. `cluster_module/` (独自高速計算エンジン)
`autttto_clustering.py` の裏側で呼び出される、計算量削減・高速化のためのモジュール群です。

大量の断層モデルペアの距離計算（カスタムコサイン距離など）を高速に処理するため、Pythonだけでなく C言語 (`.c`) や Cython (`.pyx`) を用いて記述されており、OpenMPを用いたマルチスレッド並列計算にも対応しています。（現状では出力結果の距離部分に異常があるため、正しくクラスター分割できません要改良です。）
また、コード内で安定化クラスタリング（SHC）(『階層クラスタリングの安定化』原 聡 et al.)の実装がされています。これが上記の距離異常の原因？

## 詳細な実行手順（ワークフロー）

本ディレクトリのプログラムは、すべてシェバン（Shebang）が設定されています。システム環境変数（PATH）に本ディレクトリを通しておくことで、PDTIの結果が格納された任意の作業ディレクトリから直接コマンドを呼び出して実行可能です。

### Step 1: スナップショットデータの抽出
作業ディレクトリ（`fort.40` などが存在する場所）にて、以下のコマンドを実行します。
引数の `TR` (Rise time) と `st_max` (最大すべり量等) の値は、PDTI実行時の設定ファイル（`run_PDTI_TA.bash` など）の `const/` 内を参照して入力してください。


# 【単一断層面の場合】
# ※ 最後の引数は必ず 0 (Average) を指定します。
snapper.bash <TR> <st_max> 0

### Step 2: 節面の最適化（推奨）
クラスタリングを実行する前に、節面の不連続性を解消するために以下のコマンドを実行します。
```bash
align_fault.py
```

# 【複数断層面の場合】
```bash
snapper_mf.bash <TR> <st_max> 0
出力: 節面が整理・上書きされたデータが保存されます。
```

### Step 3: クラスタリングの実行
以下のコマンドでメインのクラスタリング処理を開始します。
```bash
autttto_clustering.py
```
実行すると、ターミナル上に以下のような実行モードの選択肢が表示されます。
実行モードを選択してください:
  1: 対話モードで1回実行
  2: 事前定義済み設定で複数回実行 (コード編集が必要)
  3: 全通り試行モード
選択 (1-3, デフォルト1):
通常は 1 を入力（またはそのままEnter）して「対話モード」に進みます。
その後、コンソールに表示される指示に従い、以下のようなパラメータを順次選択します。

分析に使用する変数の選択方法（例：法線ベクトル＋すべり量 など）

PCA（主成分分析）による次元圧縮の有無と累積寄与率

距離メトリクス（コサイン距離、ユークリッド距離、または独自設定）

クラスター数の決定方法（シルエットスコアを用いた自動判定、または手動指定）

【実行完了後】
処理が正常に終了すると、選択したパラメータ名が付与された新しい出力用ディレクトリ（例: result_pca90_cosine.../）が自動で作成され、その中に clusteringSnap2.dat や樹形図（デンドログラム）のPDFファイルなどが保存されます。

この出力データを用いて、2_plotting ディレクトリの各種GMTスクリプトによる可視化フェーズへと進みます。
