1_clustering: Data Extraction and Clustering

有限断層インバージョン（PDTI）の出力結果をもとに、地震波形や地殻変動から推定された断層面上の破壊過程（スナップショット）を抽出し、機械学習（階層型クラスタリング）を用いて代表的な破壊モデルを分類・抽出するパッケージです。

収録ファイル・ディレクトリの詳細

メインスクリプト

autttto_clustering.py
クラスタリング処理の司令塔となるPythonスクリプトです。ターミナル上で対話的に実行モードやパラメータ（主成分分析の有無、距離メトリクス、閾値判定手法など）を選択しながら、階層型クラスタリングを実行します。PandasやSciPy、Scikit-learnを用いて高度なデータ処理と樹形図（デンドログラム）の作成を行います。

前処理・データ成形スクリプト

snapper.bash / snapper_mf.bash
インバージョン結果（fort.40, mrf.dat など）を読み込み、クラスタリングの入力となる時空間スナップショットデータ（snap_y.dat など）を生成するシェルスクリプトです。単一断層面用（snapper.bash）と複数断層面用（_mf.bash）に分かれています。内部で bin/ フォルダにあるFortranツール（fgensnap_y 等）を呼び出します。

align_fault.py
地震のメカニズム解には常に2つの直交する節面（Nodal plane）が存在しますが、クラスタリングの精度を向上させるため、実際の断層面に近い方の節面を自動または手動で選択・揃える前処理スクリプトです。

高速計算モジュール

cluster_module/
数万～数十万に及ぶスナップショット間の距離（コサイン類似度やユークリッド距離など）を計算するための独自拡張ライブラリです。計算ボトルネックを解消するため、C言語やCython（.pyx）を用いて実装されており、サーバー環境やローカルPCでのOpenMPによる並列高速計算を可能にします。

詳細な実行手順（ワークフロー）

PDTIの計算結果（fort.40 など）が存在する作業ディレクトリにて、以下の順序でコマンドを実行します。
※各スクリプトにはシェバンが設定されているため、環境変数（PATH）に本ディレクトリを登録しておけば、どこからでもコマンド名のみで実行可能です。

Step 1: スナップショットデータの抽出

インバージョンの生データから、時間枠（Time window）ごとのすべり量やメカニズム解（法線ベクトル、すべりベクトル）を抽出します。

# 単一断層モデルの場合
snapper.bash <TR> <st_max> 0

# 複数断層モデルの場合
snapper_mf.bash <TR> <st_max> 0


<TR> と <st_max>: インバージョン実行時の設定値です。通常、作業ディレクトリの const/run_PDTI_TA.bash などに記載されています。

第3引数 (0): 0は平均値（Average）出力を意味します。クラスタリングには生データをそのまま使用するため、必ず 0 を指定してください。

Step 2: 節面の最適化（推奨）

抽出したスナップショットデータに対し、走向（Strike）・傾斜（Dip）・すべり角（Rake）の節面を適切に選択します。これを省くと、同じ物理的意味を持つ破壊が「異なるクラスター」として誤判定される可能性があります。

align_fault.py


Step 3: クラスタリングの実行

メインスクリプトを起動し、ターミナル上の指示に従って解析を進めます。

autttto_clustering.py


実行時の対話プロンプト（モード選択）

スクリプトを起動すると、以下の選択肢が表示されます。

対話モードで1回実行 (デフォルト: 1)
一つ一つのパラメータを手動で決定しながら進めるモードです。

変数の選択: 解析に使用する物理量（すべり量、ベクトル成分など）を選択します。

主成分分析 (PCA): 次元圧縮を行い、ノイズを減らすかどうかを選択します（例：累積寄与率90%でカット等）。

距離メトリクス: サンプル間の類似度をどう測るか（コサイン類似度、ユークリッド距離など）を選択します。

事前定義済み設定で複数回実行 (2)
スクリプト内に記述された設定リスト（バッチ処理）に基づき、複数の条件でクラスタリングを連続実行します。パラメータチューニング時に便利です。

全通り試行モード (3)
実装されている手法の組み合わせを網羅的に計算します。

クラスター数の決定

階層型クラスタリングの過程で、最適なクラスター数を決定するためのシルエットスコア（Silhouette Score）などが計算され、樹形図（デンドログラム）が描画されます。適切なクラスター数（閾値）を決定してください。

Step 4: 結果の確認

実行が完了すると、作業ディレクトリ内に clusteringSnap2.dat などの結果ファイル（各スナップショットにクラスター番号のラベルが付与されたデータ）が出力されます。
このデータを用いて、2_plotting ディレクトリのGMTスクリプトで可視化を行います。
